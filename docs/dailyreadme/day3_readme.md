# Day 3 – Data Transformation & Curation (Glue ETL)

## Objective
Convert raw CMS Open Payments CSV data into clean, typed, analytics-ready datasets using AWS Glue ETL and Parquet.

This day focuses on schema enforcement, data quality, and performance optimization.

---

## What Was Completed
- Curated S3 zone created
- Glue database: open_payments_curated
- Glue ETL job using Spark (Glue 4.0)
- Explicit schema selection and type casting
- Deduplication and Parquet output
- Athena validation on curated data

---

## Outcome
- CSV ambiguity removed
- Strong schema enforced
- Analytics-ready Parquet datasets produced
